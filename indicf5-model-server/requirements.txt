# --- model & audio (heavy; pulled only in the model-server image) ---
# torch is large; the exact build (CPU/CUDA) is chosen in the Dockerfile.
# Version pins mirror IndicF5's own requirements
# (github.com/ai4bharat/IndicF5/blob/main/requirements.txt):
#   - transformers<4.50: newer versions default to "meta device" lazy loading,
#     which IndicF5's custom model code doesn't support (RuntimeError: ...
#     expected device meta!).
#   - numpy<=1.26.4: IndicF5 is not compatible with numpy 2.x.
# torch/torchaudio are pulled transitively by IndicF5 (via f5-tts) and pinned to
# a specific version in constraints.txt (applied in the Dockerfile with `-c`).
# They are NOT listed here directly — pinning the same package in both a
# requirements file and a constraints file makes pip's resolver fail with a
# "duplicate pin" conflict. This image targets linux/arm64 (Apple Silicon),
# whose default PyPI torch wheels are already CPU-only, so no special index.
transformers>=4.44,<4.50
huggingface-hub>=0.24,<0.26
soundfile>=0.12
numpy<=1.26.4
